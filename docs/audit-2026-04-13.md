# Whiskcam — Audit Complet du Site
**Date** : 13 avril 2026
**Framework** : Awesome DESIGN.md (Google Stitch) + benchmarks Apple / Stripe / Tesla
**Score global** : **7.8 / 10** (vs 7.3 avant optimisation copywriting)

---

## 1. Visual Theme & Atmosphere

### Etat actuel
- **Font** : DM Sans (Google Fonts) — propre, lisible, moderne. Bon choix pour D2C pet/lifestyle.
- **Couleur accent** : Amber (#F5A623) — chaud, amical, cohérent avec l'univers animal. Une seule couleur accent (comme Apple, Stripe, Tesla).
- **Palette neutre** : wk-black (#1A1A1A) + 6 niveaux de gris bien définis + wk-warm (#F7F7F5) pour les fonds alternés.
- **Atmosphere** : Le hero vidéo crée une accroche émotionnelle immédiate. L'alternance fond blanc / fond chaud suit le principe "cinematic pacing" d'Apple.
- **Overlay** : 50% noir sur la vidéo hero — bon compromis lisibilité/immersion.

### Score : 8/10
**Forces** : Palette disciplinée (1 accent), atmosphère chaude, vidéo hero immersive.
**Faiblesses** : Le design manque de "tension" — tout est rond, doux, prévisible. Apple crée du contraste avec des sections noires/blanches alternées. Whiskcam reste trop dans le beige/blanc homogène après le hero.

### Recommandation
Ajouter une section sombre (bg-wk-dark ou bg-wk-black) dans le funnel pour casser la monotonie. Idéalement la ComparisonTable ou le CTA Banner (ce dernier l'a déjà avec l'image de fond).

## 3. Typography

| Role | Taille | Poids | Line-height | Usage |
|------|--------|-------|-------------|-------|
| Hero title | clamp(2rem, 8vw, 4.5rem) | bold | 1.05 | Titre principal |
| Section heading | clamp(1.75rem, 3vw, 2.5rem) | bold | 1.2 | Titres sections |
| Body | 15px-18px | normal | relaxed | Texte courant |
| Overline | 11-12px | semibold | normal | Labels sections |
| Trust strip | 11px | medium | normal | Badges confiance |
| Button | inherits | inherits | inherits | CTA |

### Score : 7/10
**Forces** : Hiérarchie claire hero → sections → body. Clamp() pour le responsive est la bonne approche.
**Faiblesses** :
- **Pas de letter-spacing négatif** sur les headlines (Apple utilise -0.28px à 56px, Stripe -1.4px). Ça manque de "serrage" premium.
- **"Bold" partout pour les titres** — Stripe utilise weight 300 pour ses headlines (confidence silencieuse). Whiskcam pourrait tester semibold (600) au lieu de bold (700) pour les section headings.
- **L'overline à 11px** est presque trop petit sur certains écrans. 12px serait le minimum lisible.

### Recommandation
Ajouter `letter-spacing: -0.02em` aux h1/h2 pour un feel plus premium. Tester `font-weight: 600` au lieu de `700` sur les section headings.

## 4. Component Stylings

### Boutons
- **Radius** : 8px (--radius-btn) — bon, cohérent avec Stripe (4-8px). Pas trop rond (pill), pas trop carré.
- **CTA primaire** : Amber bg, texte sombre. PROBLEME : le texte sur amber peut avoir un ratio de contraste insuffisant (à vérifier WCAG).
- **Hover** : Amber assombri — OK.
- **Sticky ATC bar** : Bien implémenté, états multiples (loading, success, disabled).

### Cards
- **Radius** : 16px (--radius-card) — TROP pour un site e-commerce premium. Apple utilise 5-12px, Stripe 4-8px, Tesla 4-12px. 16px fait "app mobile" / "playful" plutôt que "premium D2C".
- **Feature cards** : Hover lift (-translate-y-1) + shadow — bon, interactif.
- **Review cards** : Bonne structure (rating, verified badge, photos, expand).

### Navigation
- Transparent → white au scroll (50px seuil) — pattern standard bien exécuté.
- Burger mobile avec overlay plein écran — OK.
- Pas de backdrop-filter blur — manque l'effet "glass" que Apple et Tesla utilisent.

### Score : 7/10
**Forces** : Boutons bien conçus, sticky ATC solide, feature cards interactives.
**Faiblesses** :
- `border-radius: 16px` sur les cards est trop large pour un feel premium
- Pas de backdrop-filter sur la nav (le "glass effect" est maintenant standard)
- Contraste texte/fond potentiellement insuffisant sur les boutons amber

### Recommandation
- Réduire --radius-card de 16px à 12px
- Ajouter backdrop-filter: blur(12px) + fond semi-transparent sur la navbar au scroll
- Vérifier le ratio de contraste WCAG du texte sur fond amber

## 5. Layout Principles

### Spacing
- Base 8px implicite (Tailwind default).
- Sections utilisent py-10/py-12 pour le rythme vertical.
- Max-width : 5xl (64rem/1024px) pour la plupart des sections, 6xl pour le hero (1152px). Raisonnable.

### Grid
- Features : 1col → 2col → 3col — standard et efficace.
- Product : 1col → 2col (gallery + buy box) — parfait.
- Reviews : 1col → 2col → 4col — approprié.

### Whitespace
- Le site a un bon rythme vertical section par section.
- Le hero est full-viewport (100dvh) — comme Tesla.
- Les sections post-hero sont plus compactes — c'est bien pour le e-commerce (scroll = engagement).

### Score : 8/10
**Forces** : Responsive grid bien pensé, hero full-viewport, rythme section correct.
**Faiblesses** : Les sections se ressemblent trop en espacement (py-10/py-12 partout). Varier le padding pour créer un rythme plus dynamique — sections clés (Product, CTA) méritent plus d'espace.

## 8. Conversion Funnel Analysis

### Parcours actuel (post-optimisation)
```
Announcement Bar (urgence countdown) 
→ Hero (hook + CTA + urgence badge)
→ Stats + Trust Strip (crédibilité)
→ Video Showcase (preuve / désir)          ⚠ PAS DE CTA
→ Product Section (buy box)                ✅ Position 4
→ Reviews (social proof)
→ Features (éducation)                     ⚠ REDONDANT
→ How It Works (simplification)            ⚠ PAS DE VISUELS
→ Comparison (différenciation)
→ CTA Banner (relance)
→ FAQ (objections)
→ Footer
+ Sticky ATC Bar (mobile, conditionnel)
+ Email Popup (capture)
```

### Points de friction identifiés

1. **Mobile : pas de product shot dans le hero** — 60%+ du trafic e-commerce est mobile. Le visiteur voit le titre et le CTA mais ne sait pas à quoi ressemble le produit.

2. **Video Showcase sans CTA** — C'est le moment de pic émotionnel (le visiteur voit les vidéos POV incroyables) et il n'y a aucun bouton pour capitaliser sur cette émotion.

3. **Features Grid redondant avec le buy box** — Les key benefits du buy box (1080P, 24g, no app, 2h battery) sont les mêmes que les 6 features. Double emploi.

4. **How It Works sans visuels** — Étapes texte-seulement. Un visiteur qui scrolle vite ne s'arrête pas sur du texte. Les images/illustrations sont essentielles ici.

5. **Pas de "X personnes regardent" / "Y vendus" dans le buy box** — Le buy box a des trust badges mais pas de signaux de demande en temps réel.

6. **Email popup timing** — Non visible dans le code quand il se déclenche. S'il apparaît trop tôt, il interrompt le funnel. S'il apparaît trop tard, le visiteur est parti.
